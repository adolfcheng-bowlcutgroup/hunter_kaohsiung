本機測試版：電商資料全域切換

更新內容：
1. 「加入電商資料」改為固定在右下角的懸浮按鈕。
2. 啟用後會同時影響：
   - 商品日銷售趨勢
   - 單一商品趨勢分析
3. 單一商品趨勢分析中，現場統計卡片永遠只依現場資料計算，不會因為加入電商資料而改變。
4. 電商報表目前只有數量，金額採用「現場含稅銷售金額 ÷ 現場銷售數量」回推單價：
   - 若同商品同日期有現場銷售，使用該日現場單價。
   - 若同商品同日期沒有現場銷售，使用該商品整段期間現場平均單價。
5. 商品日銷售趨勢啟用電商後：
   - 數量欄位 = 現場數量 + 電商數量。
   - 金額欄位 = 現場含稅金額 + 電商回推含稅金額。
   - 有加入電商的格子會以淡紅底標示。
6. 單一商品趨勢分析啟用電商後：
   - 藍線：現場銷售數量。
   - 紅線：電商銷售數量。
   - 明細表新增電商數量、電商回推金額、合併數量、合併回推金額。

本機測試建議：
cd hunter_kaohsiung-main
python -m http.server 8000
然後開啟 http://localhost:8000
